# 卫生投诉分析平台 — 操作手册

## 平台简介

本平台是一个 **AI 驱动的卫生投诉数据自动化分析系统**，覆盖从数据上传 → AI 打标签 → 交叉统计分析 → 报告生成的全流程。

访问地址：`https://complaint-analysis-production.up.railway.app`

---

## 第一步：注册与登录

1. 打开平台地址，点击 **「立即注册」**
2. 输入用户名和密码（至少 4 位），点击 **「注册」**
3. 注册成功后跳转登录页，输入账号密码登录

---

## 第二步：配置 AI 大模型

> 平台不自带 AI Key，**每个用户必须配置自己的 API Key**。

1. 登录后，点击右上角 **「AI 设置」** 按钮
2. 在弹窗中：
   - **快捷选择**：下拉选择你使用的平台（DeepSeek / 通义千问 / 智谱 GLM 等），会自动填入 API 地址和模型名
   - **API Key**：填入你平台提供的密钥
3. 点击 **「保存」**

配置成功后右上角「AI 设置」旁边会显示绿色圆点。

---

## 第三步：创建项目

1. 登录后进入「我的项目」页面
2. 点击 **「新建项目」**
3. 输入项目名称（如 "2024年上半年投诉分析"）
4. 点击项目卡片进入项目详情

---

## 第四步：上传数据

在「数据与标签」页面，有两个标签页：

### 上传 Excel 数据

1. 点击 **「上传 Excel」** 按钮，选择你的数据文件（支持 .xlsx / .xls，最大 500MB）
2. 如果 Excel 有多个 Sheet，会自动弹出 Sheet 选择器，选好后点 **「确认」**
3. 上传后在下方表格可以看到文件列表和行数
4. 点击文件名可预览数据（前 200 行）

### 上传提示词文档

1. 切换到 **「提示词文档」** 标签页
2. 上传 Word 文档（.docx），其中定义标签体系和打标规则

> 示例提示词内容：
> ```
> 1. 服务态度-态度恶劣：投诉中对服务人员态度差、语气不善的描述
> 2. 服务态度-推诿扯皮：投诉中反映被推卸责任、不解决问题的描述
> 3. 收费问题-乱收费：投诉中反映收费不透明、多收费用的描述
> 4. 医疗质量-误诊：投诉中反映诊断错误、延误治疗的描述
> ```

---

## 第五步：AI 标注分析

1. 上传数据后点击 **「开始 AI 标签分析」**，或从左侧菜单进入 **「AI 标注」**
2. 在标注页面：
   - **数据文件**：选择已上传的 Excel
   - **提示词文档**：选择已上传的 Word
   - **分析字段**：选择 Excel 中要分析的文本列
   - **并发数**：选择同时调用 AI 的线程数（推荐 3）
3. 点击 **「启动 AI 标注」**，系统开始逐条分析
4. 页面实时显示：
   - 进度条（已完成/总数）
   - 每条数据的标签和 AI 思考过程
   - 错误信息（如有）
5. 可随时 **「暂停」** / **「继续」** / **「取消任务」**
6. 完成后点击 **「下载标注结果」** 获取带标签和思考过程的 Excel

---

## 第六步：标签二值化

1. 进入 **「二值化」** 页面
2. 选择数据源：
   - **前期标注结果**：选择已完成的标注任务
   - **本地上传**：上传标注好的 Excel
3. 点击 **「执行二值化」**
4. 预览窗口展示 0/1 矩阵（1=打上该标签，0=未打上）
5. 点击 **「下载」** 获取二值化文件

---

## 第七步：数据分析（SPSS 定制表）

1. 进入 **「数据分析」** 页面
2. 选择数据源（标注任务 或 上传文件）
3. **左侧变量池**显示所有可用变量：
   - 普通变量（原始数据列）
   - 0-1 标签（可组成多选题集）
4. 点击变量旁边的 **「行」「列」「分层」** 按钮，将变量放入对应区域
5. **多重响应集**：输入集合名称 → 多选子项 → 添加到行/列
6. **右侧统计指标**：勾选需要的指标
   - ☑ 频数：每条标签出现的次数
   - ☑ 行百分比：该格频数 ÷ 行合计 × 100
   - ☑ 列百分比：该格频数 ÷ 列合计 × 100
   - ☑ 总计百分比：该格频数 ÷ 总记录数 × 100
   - ☑ 卡方检验：χ² / p / Cramér's V（仅单行×单列时）
7. 点击 **「运行分析」**，查看：
   - **频率表**：一个变量下各标签的频数和涉及比例
   - **交叉表**：行变量 × 列变量的交叉统计
   - **AI 分析摘要**：自然语言解读
8. 点击数值可查看该数据的**计算规则**

---

## 第八步：报告分析

报告分析页面提供多种分析方式：

### 方式一：生成 Word 报告

1. 选择标注任务
2. 上传 Word 报告模板
3. 点击 **「生成报告」** → AI 按模板风格撰写报告 → 下载

### 方式二：AI 自然语言分析

1. 选择标注任务
2. 在文本框输入分析指令，例如：
   > "分析各类投诉的占比，找出最突出的问题类型并给出改进建议"
3. 点击 **「发送分析」** → AI 返回分析结果

### 方式三：自由表格分析

1. 点击 **「新建分析面板」**
2. 从 Excel 复制表格 → 粘贴到输入框 → 自动识别为表格
3. 输入分析指令，例如：
   > "计算每个分类的频数和百分比，按降序排列"
4. 点击 **「发送分析」** → AI 返回结果
5. 可添加多个面板，互不干扰，分析完成后自动折叠

## 常见问题

| 问题 | 解决方法 |
|---|---|
| AI 调用失败 | 检查 AI 设置中的 API Key 是否正确，是否有余额 |
| 标注到一半停了 | 检查网络，点击「继续」恢复；或点「取消任务」放弃后重新开始 |
| 数据分析结果全为 0 | 确认上传的 Excel 包含「AI标签」列 |
| 页面打不开 | 用 Ctrl+F5 强制刷新，或检查 Railway 部署状态 |
| 标签为中文逗号间隔 | 系统兼容中文逗号（，）和英文逗号（,） |
| 文件太大上传慢 | 已优化为流式上传（4MB 分块），支持最大 500MB |
| 每个环节的文件能不能删 | 可以，上传区下方文件列表右侧有 ❌ 删除按钮 |

## 注意事项

1. **不要共享账号**：每个人的 AI Key 是独立的，用谁的 Key 就扣谁的钱
2. **标签定义格式**：一级标签 + 横杠 + 二级标签，如 `服务态度-态度恶劣`，数据分析会自动识别层级
3. **不要关闭服务器窗口**：本地运行时两个 PowerShell 窗口需保持打开
4. **数据持久化**：Railway 线上版使用 PostgreSQL，重启不丢数据；本地 SQLite 存储在 `backend/data/` 目录
5. **并发数建议**：DeepSeek 等国内模型建议 3~5 并发，太多可能触发限流
